Modulo 17

Internet Server

Nel corso delle lezioni in aula potranno essere visti e provati i seguenti server e le relative configurazioni:

  • DNS: BIND (Berkeley Internet Name Domain).
    Approfondimento bind.
  • Mail: Postfix (Mail Transport Agent).
    Approfondimento e-mail.
  • DataBase: MySQL .
    Approfondimento MySql.
  • Web: Apache
    Approfondimento Apache.
  • proxy: Squid.
    Approfondimento Squid.
  • list: Mailman (solo Unix)
  • newsgroup: INN (solo Unix)
  • ftp: no grazie. Meglio SSH.
    Un ottimo client per macchine M$ lo puoi scaricare da WinSCP. Provalo! Per sicurezza configuralo in modo che scelga la versione 2 del protocollo e se ti sei stancato di digitare la password prova a scoprire come si fa a generare la coppia di chiavi e dove/come vanno installate e quali permission devo avere.
    Un ssh client per macchine M$ lo puoi scaricare da SSH Client. Se non vuoi installare nulla dal lato client allora usa dal lato server MindTerm un applet java. È installato e configurato su questo sistema e se vuoi usarlo chiedi un account al tuo tutor e poi prova a connetterti (attenzione che l'applet è 600KB).
  • gestione versioni: CVS
    Questo programma è utilizzato per memorizzare i sorgenti dei programmi e quindi a gestirne le versioni. Quasi tutto il software Open Source è memorizzato in server che utilizzano CVS. Può essere usato anche per memorizzare documentazione o file di configurazione ma solo in formato testo. Esistono dei client per la consultazione del repository. Uno dei più famosi per M$ è WinCVS. Per Linux esiste invece Cervisia ma in generale è preferibile l'interfaccia browser CVSWeb.
