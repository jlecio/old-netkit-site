Modulo 1

Infrastrutture informatiche all'interno di un istituto scolastico

  • GnuPG
    Installa GnuPG e provalo usando un http keyserver italiano! Questo software è Open Source (anche se spesso si trovano versioni freeware o evaluate). Nel caso tu voglia cimentarti nell'uso dell'interfaccia a caratteri ho trovato chiare le indicazioni di badpenguin.
  • OpenSSL
    Il certificato X.509 usato dal web server a cui sei connesso è stato generato proprio con OpenSSL e permette quindi la comunicazione cifrata SSLv2 o SSLv3/TLSv1 tra client (browser) e server (web server). Con questo software si può generare un certificato digitale X.509 anche per il lato client (browser) in modo da autenticare non solo il lato server, attraverso i certificati già memorizzati nei browser, ma anche il lato client. Per trovare il riferimento a delle ottime slide che illustrano le varie configurazioni di cifratura vai a modulo 17. OpenSSL mette a disposizione gli strumenti informatici per realizzare una (Open Source) Public-Key Infrastructure (OS)PKI.
  • Stunnel
    Questo programma viene usato per costruire tunnel cifrati SSL tra client remoti e server locali/remoti quali ad esempio POP-2, POP-3, IMAP, NNTP, SMTP e HTTP. Controlla il corretto nuovo numero di porta. Questo programma ci sarà utile quando organizzeremo la raccolta di tutti i log delle macchine della rete di sperimentazione C2 in un unico server di management. Nel caso in cui tu voglia fare un test di tunneling di una sessione Virtual Network Computing (VNC) tra un Linux server e un Windows 2000 client può esserti utile questo HowTo.
