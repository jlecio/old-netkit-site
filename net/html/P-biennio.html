Il progetto

Il progetto P-biennio parte dalle indicazioni del MIUR sulla introduzione dell'Open Source all'interno della infrastruttura scolastica (Osservatorio Tecnologico) e dalla direttiva del Ministro per l'innovazione e le tecnologie Stanca (Ministro per l'innovazione e le tecnologie(MIT). Il progetto P-biennio è interamente finanziato dall'Istituto "C.Zuccante" di Mestre-Venezia.

Gli Obiettivi
  • installare un sistema di autenticazione e di condivisione dell'informazione per la sede staccata dell'IT "C.Zuccante" di Mestre/Venezia collegato al sistema di autenticazione della sede centrale.
  • standardizzare (per le varie piattaforme in uso) e regolarizzare le procedure di costruzione degli account ad ogni inizio anno
  • testare la soluzione per un successiva estensione all'intero istituto

Descrizione

Il progetto consiste nella messa in opera di un sistema operante su piattaforma GNU/Linux con distribuzione Debian Sarge che fungerà da sistema autenticazione, autorizzazione e condivisione file mediante l'uso di software Open Source. Le macchine dei laboratori del biennio verranno configurate per l'accreditamento e la condivisione di file esclusivamente a questo sistema.
Il sistema in questione sarà configurato con un servizio primario presente in sede centrale e con almeno due servizi di backup presenti nella sede staccata. L'autorizzazione all'uso delle macchine (Windows o Linux) dei laboratori avverrà attraverso la consegna di una unica coppia (utente, password) a cui corrisponderà un unico spazio disco (e le eventuali altre condivisioni configurate).
Le operazioni coinvolgeranno solo alcuni client ragruppati per tipologia di hardware/software. Solo al termine delle lezioni a giugno verranno tutti convertiti al nuovo sistema.

Hardware

Per non intaccare l'operatività del primario già esistente in LASA abbiamo installato un secondo primario su una macchina gentilmente prestata dal LASA:

  • Primario: P4
  • Secondario: PII
  • Proxy per la navigazione WEB
  • firewall con VPN con il biennio e area DMZ
  • router1: connettività a LASA (dopo 15 giugno)
  • router2: connettività ad OEN1

Stato del progetto

Alla data del 7 giugno sono stati installati e resi operanti i seguenti sistemi:

  • sede
    • Firewall configurato con area DMZ. Nell'area DMZ è operativo un servizio WEB e un servizio DNS esterno.
    • Server primario autentica
    • Server secondario autentica
    • Proxy web
    • Server DNS interno
    • router per la connessione del laboratorio LASA
    • router per la connessione del laboratorio OEN1
  • Biennio
    • Firewall configurato con area DMZ.
    • Server autentica secondario
    • Proxy web
    • router per la connessione del laboratorio LAM1

Le due sedi sono dal 7 giugno 2005 collegate permanentemente da una VPN.
